### 문제

당신은 온라인 게임을 운영하고 있습니다. 같은 시간대에 게임을 이용하는 사람이 `m`명 늘어날 때마다 서버 1대가 추가로 필요합니다. 어느 시간대의 이용자가 `m`명 미만이라면, 서버 증설이 필요하지 않습니다. 어느 시간대의 이용자가 `n x m`명 이상 `(n + 1) x m`명 미만이라면 최소 `n`대의 증설된 서버가 운영 중이어야 합니다. 한 번 증설한 서버는 `k`시간 동안 운영하고 그 이후에는 반납합니다. 예를 들어, `k` = 5 일 때 10시에 증설한 서버는 10 ~ 15시에만 운영됩니다.

하루 동안 모든 게임 이용자가 게임을 하기 위해 서버를 최소 몇 번 증설해야 하는지 알고 싶습니다. 같은 시간대에 서버를 x대 증설했다면 해당 시간대의 증설 횟수는 x회입니다.

다음은 `m` = 3, `k` = 5 일 때의 시간대별 증설된 서버의 수와 증설 횟수 예시입니다.

모든 게임 이용자를 감당하기 위해 최소 7번 서버를 증설해야 하며, 이보다 적은 수의 서버 증설로는 모든 게임 이용자를 감당할 수 없습니다.

0시에서 23시까지의 시간대별 게임 이용자의 수를 나타내는 1차원 정수 배열 `players`, 서버 한 대로 감당할 수 있는 최대 이용자의 수를 나타내는 정수 `m`, 서버 한 대가 운영 가능한 시간을 나타내는 정수 `k`가 주어집니다. 이때, 모든 게임 이용자를 감당하기 위한 최소 서버 증설 횟수를 return 하도록 solution을 완성해 주세요.

### 제한사항

- players의 길이 = 24
  - 0 ≤ players의 원소 ≤ 1,000
  - players[i]는 i시 ~ i+1시 사이의 게임 이용자의 수를 나타냅니다.
- 1 ≤ m ≤ 1,000
- 1 ≤ k ≤ 24

### 풀이 설명

✅ 각 시간대별로 사용 가능한 서버 수를 빠르게 조회하기 위해 0 ~ 23시를 key로 갖는 객체를 `available_server`로 객체로 선언하였습니다. `time`을 통해 `available_server[time]` 객체를 탐색하여 각 시간 대 별 필요한 서버의 수를 계산하고 부족한 서버가 있다면 그만큼 추가해줍니다. 서버를 한 번 증설하면 `k`시간만큼 서버가 지속되기 때문에 그 시간부터 `k`시간 동안 사용할 수 있음을 나타내기 위해(`time` ~ `time + k`)까지 `available_server` 객체 값을 수정합니다.

1. 0 ~ 23시간의 사용 가능한 서버의 수를 기록하기 위해 `available_server` 객체를 만듭니다. 모든 시간의 초기 서버 수는 0으로 설정합니다.
2. 각 시간 대의 이용자 수를 파악하여 **필요한 서버의 수**를 확인합니다.
   - 만약 필요한 서버의 수(`need_server`)보다 사용 가능한 서버의 수(`available_server[time]`)가 적다면, 필요한 만큼 서버를 증설합니다. 또한 `available_server[time]` ~ `available_server[time+k]`까지 증설된 서버의 수를 업데이트해줍니다.
   - 현재 사용 가능한 서버의 수가 이용자 수를 감당할 수 있다면, 다음 시간 대의 이용자 수를 파악하여 다시 필요한 서버의 수를 확인합니다.
